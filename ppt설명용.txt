구글 회사 설명
{
1996년 래리 페이지와 세르게이 브린이 대학원생 시절 프로젝트를 시작하여 회사 최초 검색엔진인 BackRub을 개발하였습니다.
이후 로고의 이름을 변경하여 지금의 Google이 탄생하였는데요.
그 이름의 탄생은 10의 10승의 뜻인 Googol을 철자를 잘못 쓴 결과인 Google이 나오게 된것입니다.
}

구글 클라우드 플랫폼(GCP)

내구성(리전)
완전 관리형의 시스템에 걸맞게 따로 국가마다 리전이 따로 있으며 사용하는 리전의 서버가 다운되버려도 다른 리전에 
데이터를 미리 복원하여 손실을 줄이게 할 수 있습니다.

통합
타사 회사의 제품들은 이전에 있던 데이터를 가져오는 작업에서부터 시간이 오래 걸리는 일이 빈번합니다.
다만 구글은 모든 제품이 통합되어있어 다른 제품의 사용할 수 있습니다.
그리고 외부 오픈소스와 타 회사용 API도 있습니다.
Vertex Ai를 예를 들어보면 데이터 준비, 모델 훈련, 예측 서비스, 모니터링의 모든 단계를 이 한 제품안에 있는 것입니다.

비용
모든 시스템이 완전 관리형으로 따로 서버를 업데이트 하는 불편을 사지를 않으며 인력과 서버를 따로 마련해야 하는 불편을 감수하죠
Storage 데이터 관리, IAM 사용자의 권한 내용과 특정 로그 기록 관리, Vertex Ai 등 거의 모든 제품의 자동화가
되어 있기에 관리자가 따로 필요하지 않습니다 .
리전의 위치나 데이터의 크기나 쿼리 객체를 기준으로 비용이 나가므로 현재 리전 위치의 가격을 확인해보시고 데이터, 쿼리를 관리하여 비용을 줄이는 것을 추천합니다.

구글 스토리지 설명
{
확장성 : Google Cloud Storage는 거의 무한대로 확장 가능합니다. 소형 애플리케이션 지원에서부터 엑사바이트급 대규모 시스템 구축에 이르기까지 Cloud Storage로 모두 처리할 수 있으므로 높은 확장성을 가지고 있습니다.
내구성 : 데이터를 분산 저장 하며 백업은 자동처리되며 클라우드 형태이기에 데이터가 물리적으로 손상되는 경우는 거의 없다.
일관성 :데이터를 읽어들여 모델중에 하나의 유형에 맞게 변환하고 데이터를 최신화에 맞추는 강 일관성 모델과 또는 성능을 높이는 약 일관성 모델로 바꿉니다.
가용성 : 다른 국가의 서버 리전을 사용할 수 있으며 이로 인해 기존의 리전에 데이터를 손상된다 하더라도 다른 곳에 저장한 데이터를 복구 할 수 있습니다.
	리전마다 서비스들이 다르며 언어가 제공 또한 다르다. 


비용 특징
	

	또한 트렌잭션이나 요청횟수, 데이터를 변형에 쓰이는 용도가 되지는 않아 데이터의 값들을 너무 자주 바꾸지 말고 쓰세요(빅쿼리 사용 유의 사항).
	제품의 종속성으로 인해 정책 변경으로 인한 가격 변경과 사용에 문제를 일으킬 수 있습니다(예시: 유니티 가격 정책)

보안 특징
	네트워크 형식이므로 서버 송출로 인한 오류나 공격으로 데이터의 유출이나 데이터 손상등의 문제를 일으킬 수 있습니다.
	또한 개인정보를 클라우드에 올리기에 정보 유출에 민감하다.
}
